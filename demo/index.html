Focus Manager
for AngularJS

Simple key and mouse navigation in your Angular JS application.

Focus Manager for Angular JS allows you to have complete control over the order in which elements receive focus. Focus Manager provides you with a simple interface and can easily be added to both new and existing projects. Focus Manger is designed to handle the fully constructed UIs in modern web applications (panels, popups, widgets, etc.) Here are some additional benefits to using Focus Manager:

  • Simple integration
  • ARIA compatible
  • Cross-browser support
  • No jQuery dependencies
  • Small footprint
  • Service-based plugin architecture
  • Mobile support
  • Custom focus highlighter
  • Shortcut key support (uses Mousetrap.js)

Instructions

For each section, Click on a button then use TAB or SHIFT+TAB to navigate, unless otherwise indicated.

Declare a "focus-group"

Start by declaring a focus-group on a DOM container element. (Typically this will be a DIV). The focus manager will immediately control elements within the scope of this focus-group. By default it will not look too much different than what you might expect to see from the browser's native focus manager.


    <div focus-group="">
    <button>A</button>
    <button>B</button>
    <button>C</button>
    </div>
Define the tab order

Tab order is declared by using focus-index. Try avoid using tabindex when possible. The focus manager uses them behind the scenes, and mixing the two can sometimes cause unexpected results.


    <div focus-group="">
    <button focus-index="4">A</button>
    <button focus-index="1">B</button>
    <button focus-index="3">C</button>
    <button focus-index="2">D</button>
    </div>
Control the loopback

Use focus-group-head and focus-group-tail to manage the behavior of the focus when it reaches the first or last element in a group. The "loop" value will set focus to the start of the group once it has reached the end. The "stop" value will prevent any further actions from occurring once it reaches the end of the head or tail.


    <div focus-group=""
    focus-group-head="stop" focus-group-tail="loop">
    <button>A</button>
    <button>B</button>
    <button>C</button>
    </div>
Autofocus

You can have the focus manager set the focus on an element automatically when it comes into view by using focus-autofocus. To run autofocus, once use ng-show. To run multiple times, use ng-if. If you have multiple elements with focus-autofocus, you prioritize which element can the autofocus with a numeric value. 


    <div focus-group="">
    <button focus-autofocus="1">A</button>
    <button focus-autofocus="2">B</button>
    <button>C</button>
    </div>
Groups within groups

You can gain finer control of focus by placing focus-groups within other focus-groups. Like focus elements, You can control the order of focus groups by using focus-group-index.

Focus elements will receive focus based on the following priority: First, child elements; second, child groups. The focus will continue to go deeper into other child groups until there are no more elements before it moves on to another group.


    <div focus-group="" focus-group-head="loop"
    focus-group-tail="loop">

    <div>
    <button>Start Here</button>
    ...
    </div>

    <div focus-group="" focus-group-index="2">
    <div>
    <button>...</button>
    ...
    </div>
    </div>

    <div focus-group="" focus-group-index="1">
    <div>
    <button>...</button>
    ...
    </div>

    <div focus-group="" focus-group-index="3">
    <div>
    <button>...</button>
    ...
    </div>
    </div>
    </div>

    <div>
    <button>D</button>
    ...
    </div>
    </div>

Using focus stack

When working with popups and dialogs, you will most likely want the focus to return to the origin from which the popup was initialized. The focus-stack="" directive keeps track of these elements. You can stack as many layers deep as you would like. So if you have an application where dialogs open other dialogs, focus-stack="" will help you keep the origins easily organized.


    <div class="modal" ng-if="dialog.show" focus-group="" focus-stack=""
    focus-group-head="loop" focus-group-tail="loop">
    <div class="modal-dialog">
    ...
    <div class="panel-body">
    <button focus-autofocus>...</button>
    </div>
    ...
    </div>
    </div>

Declaring shortcuts

Focus Manager supports shortcuts by using Mousetrap.js. Simply


    <div focus-group=""
    <button focus-shortcut="shift+k,ctrl+o">A</button>
    <button>B</button>
    <button>C</button>
    </div>
Handles disabled elements automatically

Focus manager handles disabled elements. Just set element attribute to disabled.


    <div focus-group="">
    <button>A</button>
    <button disabled>B</button>
    <button>C</button>
    </div>
Arrow key support

You can enable arrow keys for navigation. This is useful when creating widgets such as menus and lists.


    <script>
    angular.module('app', ['fm'])
    .controller('MyCtrl', function (focusKeyboard) {
    focusKeyboard.enableArrowKeys();
    }
    });
    </script>
Turn DIVs into selectable elements

Its easy to convert non-selectable elements (such as DIVs) into selectable elements. Just add focus-index on the element. If you leave the attribute empty focus-index, it will naturally receive focus in the order in which it is displayed. You can optionally add an order value as well, focus-index="1". Items with a focus-index defined with an order value will receive focus before those that do not.


    <div focus-group>
    <button>A</button>
    <div focus-index>B</div>
    <button>C</button>
    </div>
